Basic Commands


There are a lot of Unix commands that are useful and powerful, but for essential file manipulation, there are a few necessary commands that we'll initially require:


  • pwd: this is the command to output to the terminal your current working directory.

    ...:~$ pwd
    /home/james
    ...:~$

    The absolute path to the current working directory is displayed on the next line.

  • cd path: where cd is the command for changing directories, and path is either the absolute or relative path of the intended directory.

    ...:~$ cd CS3500
    ...:~/CS3500$

    CS3500 is a folder directly below the current working directory. This is called a relative path.

    ...:~$ cd /home/james/Dropbox/CS3500
    ...:/CS3500~$

    This is an absolute path to the same folder. Supplying an absolute path allows the user to cd to any specified directory in the system, provided the user has the appropriate privileges.

  • ls -l: the ls command lists the files and sub-directories in the current working directory. There are a number of tags which can be supplied as arguments to ls which change the output. For example, ls -l lists the files in a longer form with permissions displayed.

    ...:~$ ls
    james lost+found
    ...:~$ ls -l
    drwxr-xr-x 40 james james 4096 Feb 15 10:56 james
    drwx------ 2 root root 16384 Sep 5 21:02 lost+found

     The long argument gives much more information about the files or directories.

  • ps -aux: this is the command for displaying running processes and their process IDs. The aux argument actually consists of three separate flags. The a shows processes for all users, the u displays the process's user/owner and the x shows processes not attached to a terminal.

    ...:~$ ps -aux
    USER PID %CPU %MEM VSZ RSS TTY STAT START TIME COMMAND
    root 1 0.0 0.0 33740 3124 ? Ss 15:39 0:01 /sbin/init
    root 2 0.0 0.0 0 0 ? S 15:39 0:00 [kthreadd]
    root 3 0.0 0.0 0 0 ? S 15:39 0:00 [ksoftirqd/0]
    [etc]

  • kill -9 [pid]: this command sends an uninterceptible (the 9 flag) kill signal, which, as you might have guessed, kills the process associated with the given pid.

  • top: this command line program monitors all processes in descending order of CPU usage. For example, here is a snapshot of top running on my system:
